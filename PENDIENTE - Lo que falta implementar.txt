FASE 2 — SISTEMA DE PEDIDOS (prioridad media-alta)
---------------------------------------------------------------

LÓGICA DEL NEGOCIO (confirmar antes de implementar):
  - El ENCARGADO hace el pedido de mercadería para su local
  - El ADMIN también puede cargar un pedido para un local
  - El pedido tiene: fecha de carga + fecha de ingreso (1 o 2 días después)
  - Productos se piden en distintas unidades: unidades, fardos, cortes, palés
  - El pedido va al admin → admin confirma o rechaza (con comentario)
  - Al confirmar/rechazar el local recibe un ticket con el detalle
  - Esto NO reemplaza el sistema de la empresa, es para registrar

TABLAS NECESARIAS:
  CREATE TABLE productos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    codigo VARCHAR(50),
    nombre VARCHAR(200) NOT NULL,
    precio_lista DECIMAL(10,2),
    precio_sugerido DECIMAL(10,2),
    unidad_base VARCHAR(50),        -- "unidad", "fardo", "corte", "palé"
    cantidad_por_fardo INT,
    cantidad_por_pale INT,
    activo TINYINT DEFAULT 1,
    creado_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );

  CREATE TABLE pedidos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    local_id INT NOT NULL,
    creado_por INT NOT NULL,        -- usuario que cargó el pedido
    fecha_carga DATE NOT NULL,
    fecha_ingreso DATE NOT NULL,
    estado ENUM('PENDIENTE','CONFIRMADO','RECHAZADO') DEFAULT 'PENDIENTE',
    comentario_admin VARCHAR(500),
    creado_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (local_id) REFERENCES locales(id),
    FOREIGN KEY (creado_por) REFERENCES usuarios(id)
  );

  CREATE TABLE pedido_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    pedido_id INT NOT NULL,
    producto_id INT NOT NULL,
    cantidad DECIMAL(10,2) NOT NULL,
    unidad VARCHAR(50) NOT NULL,    -- la unidad en que se pidió
    FOREIGN KEY (pedido_id) REFERENCES pedidos(id) ON DELETE CASCADE,
    FOREIGN KEY (producto_id) REFERENCES productos(id)
  );

FLUJO ENCARGADO:
  1. Va a "Cargar pedido"
  2. Selecciona fecha de carga y fecha de ingreso
  3. Ve tabla de productos disponibles (código, nombre, precio lista, unidades)
  4. Ingresa cantidades
  5. Previsualiza ticket → confirma → envía al admin

FLUJO ADMIN:
  - Ve lista de pedidos pendientes
  - Abre cada pedido → ve detalle
  - Puede confirmar o rechazar (con comentario)
  - Puede exportar en PDF o Excel
  - También puede cargar un pedido él mismo para cualquier local

NOTA IMPORTANTE: las unidades (fardos, cortes, palés) necesitan
que el jefe pase exactamente cuántas unidades tiene cada presentación
antes de implementar. Por ejemplo: ¿cuántas unidades tiene un fardo
de Coca-Cola 500ml? ¿cuántos cortes tiene un palé? Esos datos
definen la lógica de conversión.
